# Кейс: объяснить API простыми словами

Кейс показывает, как я объясняю техническое понятие для пользователя без инженерного бэкграунда.

## Задача

Объяснить, что такое API, коротко и без потери смысла. Аудитория — пользователь ПК, который знаком с базовой терминологией, но не работает с разработкой.

## Подход

- Убрала детали про протоколы, методы и форматы данных.
- Оставила главную идею: API помогает системам обмениваться данными по правилам.
- Добавила пример из понятного пользовательского сценария.
- Сохранила точность: API — это не само приложение, а способ взаимодействия.

## Версия объяснения

API — это набор правил, по которым разные приложения обмениваются данными и командами.

Например, вы вводите адрес в приложении доставки еды. После первых букв приложение показывает список подходящих улиц. Чтобы получить этот список, оно обращается к другой системе, где хранится база адресов.

Приложение отправляет запрос: "Найди адреса, которые начинаются на эти буквы". Другая система отвечает списком вариантов. Правила, по которым этот запрос отправляется и обрабатывается, и есть API.

## Почему это работает

| Что нужно пользователю | Как решено в тексте |
|---|---|
| Понять термин | Короткое определение в первом предложении |
| Увидеть пользу | Пример с адресом в приложении доставки |
| Не утонуть в деталях | Нет лишних слов про HTTP, JSON и endpoints |
| Сохранить точность | API описан как правила взаимодействия систем |

## Где применимо

Такой формат подходит для:

- вводных статей в help center;
- onboarding-разделов для новых сотрудников;
- объяснений перед API Reference;
- материалов для нетехнических стейкхолдеров.
