# UX-портфолио

Раздел показывает, как я работаю с интерфейсными текстами: разбираю пользовательский сценарий, уточняю ограничения, пишу короткие формулировки и проверяю, помогает ли текст пройти задачу.

## Что я закрываю как UX Writer / Product Writer

- Интерфейсные тексты: кнопки, заголовки, подсказки, empty states, ошибки, алерты, баннеры.
- Пользовательские сценарии: что пользователь делает, где может ошибиться и какой следующий шаг ему нужен.
- Связь текста с продуктовой логикой: статусы, ограничения, backend-ответы, настройки профиля, help center.
- Унификация паттернов: одинаковые состояния называются одинаково в интерфейсе, задачах и документации.
- AI-assisted workflow: использую AI для вариантов формулировок и поиска пропущенных состояний, но финальное решение проверяю вручную.

## Как я работаю с UX-текстом

1. Определяю сценарий: что пользователь хочет сделать и в каком состоянии находится.
2. Уточняю ограничения: обязательность шага, данные backend, права доступа, риск потери данных.
3. Пишу несколько вариантов: короткий, развернутый, для ошибки, для подсказки, для пустого состояния.
4. Проверяю текст в макете: длина, переносы, соседние элементы, CTA, состояние после действия.
5. Сверяю с документацией: help center, FAQ, release notes и задачи должны объяснять сценарий теми же словами.

## Что важно для продукта

UX-текст должен не просто звучать дружелюбно. Он должен снижать количество тупиковых сценариев, повторных обращений в поддержку и ошибок пользователя.

Поэтому я смотрю на текст как на часть продукта: формулировка должна быть проверяемой, согласованной с логикой интерфейса и полезной для пользователя в конкретный момент.
